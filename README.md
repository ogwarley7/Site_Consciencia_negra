# Projeto: Site Estático — Consciência Negra

Resumo
-------
Site estático com 3 páginas (História, Personalidades, Desafios & Ações) feito com **HTML, CSS e JavaScript puro**.
Objetivo: exercício de semântica, acessibilidade, responsividade, micro-interações e publicação no GitHub Pages.

✔️ Requisitos técnicos implementados
- Stack: HTML + CSS + JS (sem frameworks).
- 3 páginas interligadas: `index.html`, `personalidades.html`, `desafios.html`.
- Imagens: cada página contém ao menos 3 imagens com `alt` descritivo (algumas usam links para imagens com licença livre).
- Efeitos: hover, transitions, micro-animações (`@keyframes`) e foco visível.
- Responsividade: breakpoint para ≤480px, layout fluido.
- Acessibilidade: landmarks semânticos (header, nav, main, section, article, footer), contraste razoável, foco visível, `aria-labels`.
- Performance: imagens abaixo da dobra usam `loading="lazy"`.
- SEO básico: `title`, `meta description`, headings hierárquicos (um `h1` por página).
- Licenças e créditos: ver seção **Créditos**.

Como usar / Deploy no GitHub Pages
---------------------------------
1. Crie um repositório no GitHub (ex: `consiencia-negra-site`) e faça `git push` do conteúdo deste projeto.
2. Nas configurações do repositório (Settings → Pages) selecione a branch `main` (ou `master`) e a pasta `/` (root).
3. Salve. O GitHub Pages publicará em `https://<seu-usuario>.github.io/<nome-do-repositorio>/` (pode levar alguns minutos).
4. Opcional: use `index.html` na raiz (já configurado).

Sugestões de imagens e licenças
------------------------------
As imagens nos exemplos são links para arquivos com uso tolerado para fins educacionais. Para produção:
- Prefira imagens em formatos leves (`webp`, `avif`) e otimize.
- Use bancos com licença compatível: Wikimedia Commons, Unsplash (ver termos), Pexels etc.
- Credite autores e links das imagens na seção **Créditos** abaixo.

Créditos & Fontes

- Imagens de exemplo: Wikimedia Commons (ilustrações históricas e imagens com licença livre).
